## What it manages

| Resource | Backend |
|----------|---------|
| `starfly_fabric` | `StarlightFabric` CRD (Kubernetes) |
| `starfly_mcp_tool` | `POST/DELETE /v1/mcp/tools` |
| `starfly_ssf_stream` | `POST/DELETE /v1/signals/stream` |
| `starfly_agent_identity` | `POST /v1/identity/agent` |
| `starfly_encryption_key` | `POST /v1/identity/agent/encryption-key` |

Helm chart install stays in `helm_release` — see [examples/complete](https://github.com/raygj/project-starfly-fabrics/tree/main/terraform-provider/examples/complete).
